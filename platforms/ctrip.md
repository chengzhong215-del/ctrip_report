# 携程平台差异 · Ctrip

## 什么时候读这个文件

用户选择携程作为分析平台时（单平台深度模式）或多平台对比模式包含携程时。

## 一、搜索与抓取指令

### 优先级顺序

1. `web_search("<酒店全名> 携程")` → 通常会返回 `hotels.ctrip.com` 或 `m.ctrip.com` 的主页链接
2. `web_fetch(<PC 端主页链接>)` → PC 端信息最全，优先抓
3. 如果 PC 端信息不足，补 `web_fetch(<移动端链接>)`
4. `web_search("<酒店全名> 携程 差评")` → 补差评原文
5. `web_search("<商圈名> 酒店 携程 评分")` → 找同商圈竞品（至少 2 家）
6. `web_search("<酒店全名> 钻级")` → 确认钻级状态（1-5 钻 / 金钻 / 铂钻）
7. `web_search("<酒店全名> 口碑榜 携程 2025")` 或 `...2026` → 查携程榜单荣誉

### 携程特有的数据点

这些是携程主页上能看到但其他平台可能没有的字段，尽量都抓全：

- **钻级标识**（黄色钻石图标，数量 1-5；金钻和铂钻是顶级标识）
- **国标挂牌星级**（若有，用黄色五角星显示；约 98% 的酒店没有）
- **辅助档次**（有些页面显示"豪华型""高档型""舒适型""经济型"）
- **携程口碑榜 / 亲子榜 / 商旅榜 / 高端榜**等平台榜单
- **"携程用户之选"**等季度标识
- **子项评分**：卫生 / 设施 / 环境 / 服务（这是携程标配的四子项）
- **住客印象标签云**（带次数）
- **"已经第 X 次入住"**这类老客点评
- **差评专区**（下拉可查）

## 二、携程的档次体系（务必正确表达）

**携程有三层档次系统，并行使用**：

1. **携程钻级**（平台自评）：1-5 钻 + **金钻** + **铂钻**。图标是黄色钻石
2. **国标星级**（国家旅游局/文旅部评定，挂牌）：1-5 星 + 白金五星。图标是黄色五角星
3. **辅助档次**（平台给出的文字描述）：豪华型 / 高档型 / 舒适型 / 经济型

### 关键事实

- **约 98% 的在线酒店没有挂牌国标星级**（全国仅 6468 家挂牌，五星级 739 家）
- **钻级 ≠ 星级**：厦门、广州等地监管部门曾明确要求区分。混用是典型外行
- 钻级的评定综合了"历史成交价、设施丰富度、消费者认可分"等因素
- 新酒店开业 6 个月内可申请调整钻级；平台日常监控，可升可降
- **2017 年携程曾废除 1-2 钻体系**，把平台产品等级简化为 3 钻以上，但酒店钻级仍保留 1-5 钻（搜索时要注意这个历史变化）
- **携程会员等级**（普卡/白银/黄金/铂金/钻石/金钻/黑钻）和酒店钻级是**两回事**，别混淆——这个是用户端会员，不是酒店档次

### 报告里的标准写法

**单纯携程分析时**：

> 该酒店在携程显示为**五钻**（平台自评），**国标五星**（文旅部挂牌），同时标记为"豪华型"。

**如果酒店没挂牌国标星**：

> 该酒店携程钻级为四钻（平台自评），未挂牌国标星级（符合全国约 98% 在线酒店的实际情况，携程钻级是其主要参考标识）。

**完全错误的写法**：

> 该酒店是五星级酒店 ← **错**，没挂牌就不能这么写
> 该酒店为五钻酒店（= 五星级）← **错**，两者独立

## 三、携程评分体系

- 总分：1-5 分，小数位到 0.1
- 子项：卫生 / 设施 / 环境 / 服务（四项）
- 时间权重：近期点评权重高
- 折叠机制：携程会折叠"底质"点评（无帮助性的短评）

### 判断绝对值 + 相对值

- 4.0-4.3：中端酒店中位数
- 4.4-4.6：同档次里属于"合格偏好"
- 4.7-4.8：同档次里属于"突出"
- 4.9-5.0：顶尖（但要警惕是否点评数太少造成的偏高）

但**这些只是参考**，绝对值不如相对值重要。要看同商圈同档次的平均水平。

### 携程"超棒"标签的历史变化

2024 年之前：4.5+ 可获"超棒"标签
2024 年后：携程有调整过规则（降低门槛以缓解"超棒焦虑"）

写报告时不要把 4.5 当作"优秀"的死线，而要看**同商圈同档次的相对位置**。

## 四、携程的图片展示规则

- **一级页面首屏露出约 7 张**（这是版面设计，不是酒店只上传了 7 张）
- **相册总数**：点击"酒店相册"或下拉进二级页面后可见全部，通常数十张
- **高档酒店相册**可能上百张
- **C 位**：首屏前 3 张影响最大（列表页 + 详情页首屏）

**报告必须明确区分"首屏露出"和"相册总数"**：

> 该酒店携程一级页面首屏露出 7 张（版面标配），相册总数 43 张。C 位前 3 张为 <外立面/大堂/标准房>，未触及 <江景套房/粤菜厅> 等差异化资产。

## 五、携程的榜单与独家标识

写报告时若酒店获得以下标识，都要列出：

| 标识 | 含义 |
|---|---|
| 携程口碑榜 | 综合口碑年度榜单 |
| 亲子榜 | 亲子友好酒店 |
| 商旅榜 | 商务差旅首选 |
| 高端榜 | 高端奢华酒店 |
| 金钻 / 铂钻 | 顶级钻级标识 |
| 携程用户之选 | 季度认可 |
| 已连续 X 年上榜 | 稳定性信号 |

**没上榜** 也是一种信号——一家四钻酒店完全没拿到任何平台榜单，本身说明运营侧没有发力。

## 六、差评归因时的携程特色

- 携程差评容易集中在"设施老化""空调异响""地毯发黄"等**老店痛点**——老酒店和装修后未及时升级的酒店高发
- "早餐差""早餐排队"是**连锁五星酒店**高频问题
- "卫生问题"出现一次就会影响排名（携程算法对卫生子项给高权重）
- "前台态度"是**中端酒店**高频问题
- 老客复购点评是高价值信号（"已经第三次入住""每年都来"）——看主页是否置顶过

## 七、对报告第二层诊断的帮助

在"转化漏斗诊断"时，携程侧要特别关注：

- **曝光**：有没有上榜、钻级是否和评分匹配、搜索排序可能的位置
- **点击**：首屏前 3 张图、档次标识是否完整、简介前 2 句能不能抓住
- **浏览**：相册是否有"场景完整度"（外观/大堂/客房/餐厅/公区/差异化资产 至少 5-6 类覆盖）
- **下单**：评分子项里最低分的那个是否影响关键客群、差评占比是否超过警戒线（>4% 算高）

## 八、携程侧杠杆动作的典型候选

高优（30 天）：
- 相册补至 30+ 张，C 位重新排序
- 简介重写，加客群、加情绪、加差异化资产
- 特色标签从泛化改为锋利（"中式餐厅"→"广府粤菜·黑珍珠入围"）

中优（3-6 个月）：
- 邀评机制重建（针对点评数远低于应得水平）
- 老客故事采集并做成主页置顶点评
- 差评类型归因后做专项改善（厨房/前台/客房的特定动线）

## 九、关于 HOS-like 的携程端指标

携程后台对酒店商家也有运营健康度评估（类似美团 HOS），但前台不可见，也没有统一公开名称。报告里**不要编造**"携程 HOS"这种概念——**这是美团的专有名词**。

可以这么写：

> 携程酒店后台也对商家运营健康度做综合评估（涵盖订单响应率、点评回复率、活动报名积极度等），但该指标前台用户不可见，本报告不涉及后台数据。
