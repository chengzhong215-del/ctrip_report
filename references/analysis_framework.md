# 三层漏斗分析框架 · Analysis Framework

## 为什么是三层漏斗

付费客户见过的咨询报告太多，一眼就能分辨出两种失败：

**失败一：数据陈列型**
抓完主页信息分类列出来，"主图 7 张、评分 4.5、点评 3881 条、差评占比 2.8%……"然后呢？没有然后。客户看完会想：这些我自己也能看到，你值多少钱？

**失败二：凭空推理型**
在没有足够事实的前提下抛判断——"可能是因为定位不清""估计流量被分流了""或许存在客群错位"。客户会想：你到底是在分析还是在猜？

**解法**：把报告切成严格有序的三层，每一层只做一件事，每一层的结论必须能**向下追溯**到上一层的事实。

```
第一层 · 可观察层（WHAT）
    ↓ 证据支持（可追溯）
第二层 · 诊断层（WHY）
    ↓ 指向撬动点
第三层 · 杠杆层（HOW）
```

**主轴规则**（贯穿三层，最重要）：
- 第二层的每条判断 **必须引用** 第一层的至少 1 个具体数字或事实
- 第三层的每条建议 **必须对应** 第二层的至少 1 条判断
- 违反此规则的判断和建议一律删除

## 第一层 · 可观察层（What Is There）

**只做一件事**：客观陈列这家酒店在所选平台上**能看到什么**。

### 允许出现

- 数字：评分、点评数、图片数、简介字数、竞品排位
- 事实：档次标识（按平台规则多轨标注）、榜单归属、标签列表、设施清单、距离参数
- 点评原文摘录（加引号、标明好评/差评）
- 相对位置表格（同商圈同档次竞品的横向对比）

### 禁止出现

- **评价词**："好""差""不错""优秀""一般""尚可""糟糕"
- **判断词**："建议""应该""需要""问题是""优势在于"
- **推理副词**："由此看来""显然""可以推断"
- **隐含评价的副词**："仅""只有""竟然""居然""明显"（这些词携带判断色彩）

### 第一层内容清单

每份报告第一层必须包含以下模块（按数据充足度取舍，参考 `data_quality_gate.md`）：

**1.1 基础档案**
- 酒店全名、开业年份、最近装修年份、客房总数
- 所在平台档次标识（多轨并存）
- 所属集团 / 品牌

**1.2 评分与点评**
- 总分 + 各子项评分
- 点评总数、差评数（或差评占比）
- 至少 5 条点评原文摘录（含好评、差评，区分标注）
- 住客印象标签云（高频词及次数）

**1.3 主页表达**
- 首屏图片数 + 相册总数
- C 位前 3 张图片内容
- 简介原文摘录（前 3-5 句）
- 酒店特色标签列表

**1.4 设施信息**
- 公区 / 客房 / 餐饮 / 会议 / 亲子 / 安保 六类分别列出

**1.5 位置参数**
- 最近地铁 / 机场 / 火车站距离
- 核心景点 / 商圈距离

**1.6 同商圈同档次横向对比**（≥2 家竞品）
- 评分、点评数、档次（按所选平台规则）、装修年份、平台榜单荣誉

**1.7 平台独家信号**
- 按所选平台读 `platforms/<name>.md` 补充（如携程钻级、美团 HOS、大众点评必住榜、飞猪信用住等）

### 第一层的"证据强度打分"（16 分制）

这个分数决定第二层能写多重、多少判断。也是 data_quality_gate.md 的依据。打完分写在第一层末尾。

| 维度 | 2 分（充足） | 1 分（勉强） | 0 分（不足） |
|---|---|---|---|
| 点评总数 | ≥500 | 100-500 | <100 |
| 相册图片数 | ≥30 | 10-30 | <10 |
| 简介字数 | ≥150 | 50-150 | <50 |
| 可查差评文本 | 3 条+ | 1-2 条 | 只有数字 |
| 设施列表 | 六类齐全 | 部分 | 寥寥 |
| 竞品可比 | ≥2 家同商圈同档次 | 1 家 | 找不到 |
| 位置参数 | 地铁/景点/机场/火车站四项齐 | 部分 | 只有地址 |
| 平台独家信号 | 清晰 | 模糊 | 无 |

## 第二层 · 诊断层（Why It's This Way）

**只做一件事**：基于第一层事实，解释**为什么是这样** + **这意味着什么**。

### 允许出现

- 判断、解释、对照
- 引用第一层具体数字（**必须，且必须在同一段内**）
- 横向对比产生的洞见（"A 比 B 高 X 分但点评数少 Y 倍，这意味着..."）
- 带锋利度的判断色彩（"这是一家被位置宠坏的酒店"——只要能回溯到第一层事实，允许）

### 禁止出现

- **软弱词**："可能""大概""估计""或许""似乎""貌似"——要判断就判断，不判断就删
- **无依据断言**：任何一句话读下去没有具体数字或事实支撑
- **循环论证**：A 因为 B、B 又因为 A 的结构
- **咨询黑话**：赋能、闭环、抓手、底层逻辑、价值重塑、数字化转型、生态化反——出现一次就是跑偏

### 第二层的四个必答问题

不一定每份报告都要把这四问全答，但**总得回答其中至少三个**：

**2.1 转化漏斗诊断**：这家酒店的 OTA 流量漏斗在哪里漏水？

漏斗四段：曝光 → 点击 → 浏览 → 下单。每段的漏水信号分别是：

| 阶段 | 漏水信号 | 证据来源（第一层） |
|---|---|---|
| 曝光不足 | 搜索排名低 / 不上榜单 / 无平台标识加成 | 是否上必住榜、有无特色标签、档次标识是否完整 |
| 点击率低 | 首屏图片 C 位配置失当、主图数偏少、封面没讲清差异 | 首屏图片数 + C 位内容 |
| 浏览流失 | 简介招商腔、设施列表有空洞、标签和主图不匹配 | 简介原文 + 设施清单 + 标签 |
| 下单犹豫 | 点评分低 / 差评集中 / 子项有明显短板 / 竞品在关键子项更强 | 评分 + 竞品对比 |

**写法公式**：用第一层的 [某个具体数字] 对比 [另一个数字] → 说明 [哪个阶段在漏水] → 根源是 [某个具体原因]

**示范**：
> 该酒店携程评分 4.6 / 点评数 1239 条，同商圈同档次竞品的广州花园酒店评分 4.7 / 点评数 11234 条。点评数差近一个量级，评分却在同级——这说明流量被漏在"曝光"阶段，产品并不弱。

**2.2 叙事缺口**：这家酒店的产品资产和主页表达之间的落差在哪里？

- 有什么资产（从第一层的设施清单、位置参数、独家信号里提取）
- 主页讲了什么（从第一层的简介、标签、C 位图片里提取）
- 两者错位在哪？
- 这个错位是在浪费资产 or 掩盖短板？

**写法公式**：它有 [资产 A、B、C]，但主页在讲 [D、E]，资产 A 在点评里出现 X 次却没进主图 C 位——这是 [浪费 / 失衡]

**2.3 平台匹配度**：这家酒店和所选平台的流量性格是否匹配？

参考 `platform_facts.md` 第五节，判断：

- 这家酒店的产品特征（档次、位置、品牌、客群）是什么
- 所选平台的流量性格（客群偏向、决策特征）是什么
- 两者是"天作之合"、"勉强搭配"、还是"平台错位"？

如果是多平台对比模式，这部分要展开——为什么这家酒店在 A 平台表现好 B 平台表现差，哪个平台该发力、哪个平台该减少投入。

**2.4 点评运营健康度**：从点评侧的信号看这家酒店的运营勤勉度

- 回复率（能观察到的话）
- 差评的归因结构（用第一层的差评原文来推断）
- 标签云中正向高频词是否被主页运营反向利用
- 有没有"老客再来"这类高价值点评

### 允许的判断色彩（B 档锋利风格）

这一层可以写："一家被位置宠坏的老牌五星"、"一家产品比表达跑得更快的度假酒店"——但每一句这种话**后面紧跟着必须用第一层事实解释**为什么这么说。

反例（外行写法）：
> 这是一家不错的酒店，总体来看表现可圈可点，具有一定的市场竞争力。

B 档写法：
> 这是一家被位置宠坏的老牌五星。位置 A 级（距广州塔 1.2km、距地铁 5 号线烈士陵园站 300m），评分稳在 4.5（同商圈均值 4.5），点评数 3881 条（高于同档次竞品 2-4 倍）——意味着流量靠位置自然涌入，但主页首屏仅 7 张图 C 位为外立面、大堂、标准房，未触及粤菜厅和珠江景这两项真正的差异化资产。

## 第三层 · 杠杆层（What to Do）

**只做一件事**：基于第二层诊断，给出**真能撬动生意**的动作。

### 允许出现

- 按时间窗分档的具体动作
- 每条动作必须满足"**做什么 × 谁做 × 何时做完 × 预期效果**"四要素
- 预期效果量化不了时写"需执行 X 周期后复测"或"参照同类样本"——不能凭空编数字

### 禁止出现

- **空话动词**："优化""加强""提升""改善""推进""落实""建立""完善""强化""深化"
- **空话名词**："品牌建设""用户体验""服务质量""运营效率""市场竞争力""数字化能力"
- **没对象的建议**："建议优化主页"——谁来做？
- **没时间窗**："尽快改善"——尽快是多久？
- **没预期**：必须说清做了有什么用

### 三档时间窗

**3.1 高优（30-60 天内可完成，主页运营层）**

- 重拍主图、调整 C 位、重写简介、更换标签、重构相册目录、补足首屏信息
- 执行方：通常是集团摄影团队 / 市场部 / OTA 运营岗
- 特征：**低成本、快见效、可 A/B 测试**

写法示范：
> 30 天内由集团摄影团队重拍相册至 40+ 张，粤菜厅 C 位进入首屏前 3（目前首屏为外立面、大堂、标准房）。预期效果：首页点击率提升 15-20%（参照同集团其他酒店改造后数据），30 天后复测携程钻级页面访问量和停留时长。

**3.2 中优（3-6 个月，产品/运营联动层）**

- 邀评机制重构（针对点评数远低于应得水平的问题）
- 服务动线改造（针对差评集中的问题）
- 特色产品商品化（会议包、亲子包、餐饮套餐的包装化）
- 会员体系运营（针对飞猪 / 大众点评的特殊渠道）

**3.3 低优但长期关键（半年以上，需要预算/跨部门）**

- 装修投入（针对硬件老化）
- 品牌叙事重构（针对定位模糊）
- 跨平台流量策略（针对多平台分流问题）

### 三层主轴的"可追溯链"

报告末尾必须有一张自检表：

| 建议（第三层） | 对应判断（第二层） | 判断依据（第一层） |
|---|---|---|
| 30 天重拍主图至 40+ 张，C 位补粤菜厅 | 主页未触及粤菜厅这一差异化资产 | 首屏 7 张 C 位为外立面、大堂、标准房；点评标签云中"早茶"出现 284 次 |
| ... | ... | ... |

**追不到的建议 = 凭空建议 = 必须删除**

## 一页纸总结的写法（B 档锋利风格）

报告最后一章（或精简版的结尾）用 blockquote 格式写一段**一页纸总结**，包含：

1. **一句起手式**（带判断色彩的命名）
2. **2-3 段论点展开**（但每句都必须可追到第一层事实）
3. **一句翻盘点**（点出最有杠杆的动作 + 为什么）

### 起手式提炼方法

1. 列出 3 个最突出的资产（位置/品牌/产品/客群 之一）
2. 列出 3 个最突出的漏点
3. 找资产和漏点之间的"反差"
4. 用一句带判断色彩的话总结

### 允许的句式模板（不要直接套用词组）

- "这是一家被 X 宠坏的 Y"
- "一家产品比表达跑得更快的 X"
- "一家靠 X 吃饭、靠 Y 守分、但 Z 拖后腿的 W"
- "它有 A、B、C，但主页只给了 D、E"
- "[平台] 给了它 X，但它只兑现了 Y"

### 反模式（绝对不要写）

❌ "这是一家五星级酒店" —— 这不叫结论
❌ "总体表现不错，有提升空间" —— 这叫废话
❌ "具备较强的市场竞争力" —— 这叫咨询体空话

## 多平台对比模式的特殊要求

如果用户选"多平台横向对比"，框架的重心要调整：

**第一层**：同样内容但按"平台 × 维度"二维表格展示，**直接呈现差异**

| 维度 | 携程 | 美团 | 大众点评 | 飞猪 |
|---|---|---|---|---|
| 档次标识 | 五钻 / 国标五星 | 豪华型 | 豪华型 | 豪华型 |
| 评分 | 4.5 (3881) | 4.4 (2103) | 4.5 (5612) | 4.6 (892) |
| 榜单 | 口碑榜 | 未上榜 | 2024 必住榜 | 信用住 |
| ... | ... | ... | ... | ... |

**第二层**：从"单平台诊断"变成"跨平台诊断"——重点回答：

- 同一家酒店在四个平台表达完整度的差距在哪
- 流量分流在哪（哪个平台给了最大流量但转化没做好 / 哪个平台有潜力但没发力）
- 平台匹配度应该如何分级（主打平台 / 辅助平台 / 减少投入平台）

**第三层**：建议要按平台分别给，不能混成一锅

## 报告写作流程（给 Claude 的操作指引）

写报告时，按以下顺序操作：

1. **先把第一层全抓完写完**。不要一边抓数据一边写判断——会忍不住越层
2. **打"证据强度分"**（16 分制），决定走标准版还是精简版
3. **才开始写第二层**。每写一句判断，回头到第一层找一个数字或事实支撑；找不到 → 删这句
4. **写第三层时回头确认第二层**。每写一条建议，回头到第二层找对应的判断；找不到 → 删这条
5. **写一页纸总结**。不要先起手式再找论据——先把第二层的核心诊断提炼一下，再反过来找一句起手式
6. **最后填"可追溯链"表格**作为自检
7. **对照 platform_facts.md** 过一遍所有平台术语
8. **对照 SKILL.md 末尾的自检清单** 再过一遍

这个顺序是强制的。**先写结论再找依据 = 必然越层 = 必然踩雷**。
