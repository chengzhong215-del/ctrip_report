# ctrip_report

一份 Claude Skill：为任何一家酒店生成专业的**携程主页质检报告（PDF）**，诊断店铺装修、设施水平、点评口碑、商圈竞争力四大维度，并给出可落地的分优先级优化建议。支持在酒店隶属于某集团时，追加生成**集团综合战略报告**。

---

## 这个 Skill 解决什么

酒店行业的 OTA 主页是流量转化的关键入口，但绝大多数酒店的携程主页存在"主图不够、文案空洞、标签通用、卖点浪费"等系统性问题。

本 Skill 让 Claude 能够：

1. **主动抓取**目标酒店的携程主页公开数据
2. **按五维框架**（装修/设施/点评/商圈/流量）做深度诊断
3. **输出精排 PDF**（A4，10 页左右，带页眉页脚、表格斑马纹、blockquote 强调）
4. **诚实判断数据充足性**——小酒店不硬凑 10 页，改为 5 页精简版或 1 页速评
5. **识别集团归属**——属于某集团/品牌时主动追问是否生成集团战略报告

---

## 设计原则

这份 Skill 有三条底线（写在 `SKILL.md` 顶部）：

1. **没有事实就不写判断** —— 每个结论都必须能追到携程主页/点评/竞品数据
2. **不说套话、不写空话** —— "加强品牌建设""优化用户体验"等咨询黑话永远不准出现
3. **小酒店也要保持清醒** —— 数据不足时诚实降级，不硬凑 10 页废话

## 安装与使用

### 前置依赖

```bash
# Python 依赖
pip install markdown weasyprint

# 系统字体（Ubuntu/Debian）
sudo apt-get install fonts-noto-cjk

# macOS
brew install --cask font-noto-sans-cjk font-noto-serif-cjk
```

### 作为 Claude Skill 安装

把整个 `ctrip_report/` 文件夹放到你的 Claude skills 目录下，Claude 会在符合触发条件时自动调用。

**典型触发语**（`SKILL.md` 里定义）：

- "帮我出一份携程质检报告"
- "分析一下 XX 酒店的携程主页"
- "看看这家酒店在携程上做得怎么样"
- "OTA 主页诊断"

### 手动调用 PDF 脚本

如果你只是想用里面的 Markdown → PDF 精排脚本，也可以直接：

```bash
python scripts/make_pdf.py /path/to/your_report.md
# 输出到 /path/to/your_report.pdf
```

脚本会自动从 md 第一个 `#` 标题提取名字作为页眉。

## 输出示例

这份 Skill 基于三份真实报告的分析过程提炼：

1. **广州宾馆**（4.5 分/3,881 条点评）——单店标准版
2. **广州南沙花园酒店**（4.7 分/1,239 条点评）——单店标准版
3. **岭南酒店集团**——集团综合战略报告（基于两家样本酒店的共性问题诊断）

示例报告可以在仓库的 `examples/` 目录找到（如果 fork 这个仓库时包含了示例）。

## 报告的核心框架

### 单店报告（7 章完整版）

1. 总体评分与结论（五维打分 + 一句话结论）
2. 店铺装修（主图、简介、标签）
3. 设施水平（盘点 + 结构性问题 + 差异化卖点）
4. 点评与口碑（评分结构、标签云、差评归因、运营评估）
5. 商圈竞争力（位置参数、竞品对比、客群画像）
6. 综合诊断与分优先级优化建议（高优/中优/低优）
7. 一页纸总结（blockquote 引用块）

### 集团战略报告（6 章）

1. 集团基本盘与战略背景（资产版图 + 财务现状）
2. 样本酒店质检的共性诊断
3. 集团层面问题的深层诊断（4 个根源）
4. 集团层面战略建议（方向 + 战术 + 优先级）
5. 风险与挑战
6. 一页纸总结

### 五维分析框架

- **装修与视觉呈现**（主图数量/场景/C位/简介/标签）
- **设施与产品信息**（公区/客房/餐饮/会议/亲子/安保）
- **点评口碑**（评分/标签云/差评归因/运营）
- **商圈竞争力**（位置/竞品/客群）
- **流量转化效率**（前四维度交叉诊断的派生结论）

## 数据不足时的降级规则

参见 `references/data_quality_gate.md`，核心决策：

| 数据质量分（满分16） | 处理方式 |
|---|---|
| ≥12 | 完整 7 章模板 |
| 6-11 | 精简 5 章模板 + 样本局限声明 |
| <6 | 不写报告，改为对话里的 1-2 页速评 |
