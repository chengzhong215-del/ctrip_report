## 这个 Skill 解决什么

酒店行业的 OTA 主页是流量转化的核心入口，但绝大多数酒店的主页存在系统性问题：

- **平台术语错用**——把携程钻级当五星级、把"必住榜"算到美团头上、把黑珍珠当成酒店榜单。付费客户一眼能识别出这种外行话
- **产品资产与主页表达错位**——有粤菜厅但 C 位图是通用客房、有江景却没讲清楚、有会议厅却没做商品化
- **多平台策略缺失**——同一家酒店在四个平台投入度没分级，要么四个都平庸、要么只做一个浪费其他平台的流量

这个 Skill 让 Claude 能够：

1. **主动抓取**目标酒店在用户所选平台（1-4 个）的公开数据，不凭印象、不编造
2. **严格三层分析**——第一层只陈列事实 / 第二层必须引用事实下判断 / 第三层只写真能撬动生意的动作
3. **输出精排 PDF**（A4，10 页左右，墨绿专业风）
4. **诚实判断数据充足性**——小酒店不硬凑，数据不足改为精简版或对话速评
5. **支持多平台横向对比**——直接给出"主打/辅助/减少投入"的策略判断
6. **处理缺席平台**——用户选 4 个但实际只上架 3 个时，如实标注缺席，不编数据

---

## 核心设计哲学（写在 `SKILL.md` 顶部，三条底线）

1. **没有事实就不写判断** —— 每个结论都必须能追到具体数字
2. **不说套话、不写空话** —— "赋能""闭环""抓手""数字化转型"出现一次就跑偏
3. **事实错误是致命伤** —— 付费客户会一眼识别出基本术语错误，所有平台术语严格对照 `platform_facts.md`

## 快速开始

### 前置依赖

```bash
# Python 依赖
pip install markdown weasyprint

# 系统字体（Ubuntu/Debian）
sudo apt-get install fonts-noto-cjk

# macOS
brew install --cask font-noto-sans-cjk font-noto-serif-cjk
```

### 作为 Claude Skill 使用

1. 把整个仓库目录放到你的 Claude skills 路径下
2. 在 Claude 对话里输入触发语，Skill 自动激活

**典型触发语**：

- "帮我出一份 OTA 质检报告"
- "分析一下 XX 酒店的携程/美团/大众点评/飞猪主页"
- "看看这家酒店在 OTA 上做得怎么样"
- "四平台对比这家酒店"

Claude 会：
1. 问你选哪几个平台 + 分析粒度（单平台深度 / 多平台对比）
2. 核实酒店身份（歧义时让你提供城市+商圈或主页链接）
3. 预检每个平台是否有该酒店主页
4. 按平台抓数据 → 三层分析 → 生成 md → 转 PDF 交付

### 手动调用 PDF 脚本

如果你只想用其中的 Markdown → PDF 排版脚本：

```bash
python scripts/make_pdf.py /path/to/your_report.md
# 输出到 /path/to/your_report.pdf
# 页眉日期自动取当天，页眉标题从 md 第一个 # 标题提取
```

## 报告框架：三层漏斗

### 第一层 · 可观察（WHAT）

只陈列客观事实，禁用所有评价词、判断词、推理副词。

模块：基础档案 / 评分与点评 / 主页表达 / 设施信息 / 位置参数 / 同商圈竞品对比 / 平台独家信号 / 证据强度打分（16 分制）

### 第二层 · 诊断（WHY）

必须引用第一层事实，不允许脱离证据推理。禁用软弱词、咨询黑话。

四个诊断问题：
- 转化漏斗诊断（曝光 → 点击 → 浏览 → 下单，哪段漏水）
- 叙事缺口（产品资产与主页表达的错位）
- 平台匹配度（酒店和所选平台的流量性格是否匹配）
- 点评运营健康度

### 第三层 · 杠杆（HOW）

只写真能撬动生意的动作。每条建议满足"做什么 × 谁做 × 何时做完 × 预期效果"四要素。

按时间窗分三档：
- 高优（30-60 天，主页运营层）
- 中优（3-6 个月，产品/运营联动）
- 低优但长期关键（半年以上）

### 主轴规则（强制）

- 第二层每条判断必须在同一段引用第一层具体数字
- 第三层每条建议必须对应第二层某条判断
- 报告末尾有可追溯链自检表，**追不到就删**

## 四平台事实差异（最易错的点）

| 维度 | 携程 | 美团 | 大众点评 | 飞猪 |
|---|---|---|---|---|
| 档次 | 1-5 钻 + 金钻 + 铂钻 | 豪华/高档/舒适/经济 四档 | 同美团（共享） | 同美团 |
| 特色字段 | 钻级、四子项评分 | HOS 指数、钟点房 | 必住榜、黑珍珠（餐饮） | 信用住、88VIP、会员通、F1-F6 |

**容易犯的基本错误**：

- ❌ "美团钻石酒店" → 钻石在美团是**会员等级**，不是酒店档次
- ❌ "美团必住榜" → 必住榜归属是**大众点评**
- ❌ "酒店获黑珍珠" → 黑珍珠是**餐饮榜**，酒店本身不上，只有酒店内餐厅能上
- ❌ "飞猪 F4 会员权益" + 2025/2026 年 → 2024 年 3 月后是 F1-F6 六级体系
- ❌ "主图仅 7 张" → 7 张通常是**首屏露出**，相册总数要另外查

完整事实对照表见 [`references/platform_facts.md`](references/platform_facts.md)。

## 数据不足时的降级规则

基于 16 分证据强度评分：

| 分数 | 处理方式 |
|---|---|
| ≥12 | 标准版（单平台 7 章 / 多平台对比 6 章） |
| 6-11 | 精简版 5 章 + 样本局限声明 |
| <6 | 不出 PDF，改为对话 1-2 页速评 |

详见 [`references/data_quality_gate.md`](references/data_quality_gate.md)。

## PDF 排版风格

**墨绿色主调（`#0f766e`）+ 冷静灰阶**，对标学术/专业诊断报告。

- Noto Serif CJK SC 衬线正文 + Noto Sans CJK SC 无衬线标题
- A4 精排、表格斑马纹、页码 `X / Y`、页眉自动提取标题 + 当天日期
- blockquote 一页纸总结带左侧墨绿粗线 + 大引号装饰字符
- 黑白打印友好（墨绿在灰度下仍有层次）

可在 `scripts/make_pdf.py` 里调整 CSS（搜索 `#0f766e` 改主色）。

## 使用语气（付费客户级 · B 档锋利风格）

- 像经验丰富的酒店行业顾问跟客户汇报，不是咨询公司 PPT 套话
- 保留"一句话起手式"的判断色彩（"这是一家被 X 宠坏的 Y"），但每句锋利判断后必须跟事实支撑
- 禁用"赋能""闭环""抓手""底层逻辑""价值重塑""数字化转型"等黑话
